# 当前报价规则

这份文件是当前对外报价和内部维护共同使用的汇总规则。日常优先看这一份，不需要先拆看多个子文件。

## 1. 通用原则

- 默认进入报价模式。
- 只有用户明确说导入、更新、激活、回滚、查看版本时，才进入维护模式。
- 用户说简称可以识别，但对外材质名称必须统一为：
  - `北美黑胡桃木`
  - `北美樱桃木`
  - `北美白橡木`
  - `北美白蜡木`
  - `乌拉圭玫瑰木`
- 正式报价前，只追问真正影响价格的关键参数。
- 缺参时通常一次只问一个最关键问题。
- 正式报价只给一个正式结果，不同时给多个候选正式报价。
- 正式报价一旦成立，不要在同一条回复尾部追加新的确认问题。
- 如果条件仍不完整，只能给“仅供参考”的参考报价。

## 2. 产品识别原则

- 如果用户已经明确给出 `产品编号`、`具体产品名称`，就不要重复追问本应由产品自身承载的信息。
- 如果用户已经明确给出 `具体产品名称`，也不要在正式报价后回头确认它是不是另一个相邻产品。
  例如：
  - 用户已说 `经典箱体床`，不能在正式报价后再问“是箱体床还是架式床”
  - 用户已说 `抛物线架式床`，不能在正式报价后再问“是不是其他架式床款式”
- 如果用户已经明确给出 `具体产品名称`，进入预检时要保留该产品名称本身，不要先把它改写成泛品类。
  例如：
  - `升级经典门衣柜` 不要先改写成 `衣柜`
  - `金属玻璃门书柜` 不要先改写成 `书柜`
  - `抛物线架式床` 不要先改写成 `床`
- 如果用户已经明确表达 `定制 / 订制 / 订做 / 订`，直接按定制路径处理。
- 如果用户给出的尺寸明显偏离目录标准尺寸，也直接按定制路径处理。
- `系列 + 品类` 若能唯一指向产品，也按明确产品处理，例如：
  - `流云 + 衣柜 = 流云衣柜`

查价时也遵循同样原则：

- 用户给了明确产品名，优先 `精确产品名 + 已给尺寸` 查价
- 不要先用 `模糊包含` 把结果扩成多个相邻产品
- 例如：
  - `升降桌` 不要扩成 `带屉升降桌`
  - `罗胖椅` 不要扩成 `高背罗胖椅`、`新罗胖椅`

## 3. 追问顺序

### 柜体类

- 固定顺序：`进深 > 是否带门 > 门型 > 系列`
- 如果已经命中明确产品，不再重复追问带门、门型、系列。
- 如果已经命中明确柜体产品，且目录里有标准进深，可直接按标准进深继续报价。
- 如果是泛品类 `书柜 / 衣柜 / 玄关柜 / 电视柜 / 餐边柜`，且 `长度、高度、材质` 已齐、也没提到抽屉、灯带、异形、岩板、双面门这类已知改价结构，可先按该品类默认基础档正式报价。
- 只有无法安全套用默认基础档时，才继续追问进深。

### 儿童房

- 儿童床：先确认床型，再确认尺寸和材质。
- 上下床对外叫法统一使用：
  - `挂梯款`
  - `梯柜款（梯柜下可储物）`
- 对 `定制 / 非标儿童床`、`半高床`、`高架床`、`错层床`，转入模块化儿童床路径，不和目录成品价混算。
- 半高床 / 高架床如果带床下柜体，床体与柜体拆开报价再合并，不把床下衣柜套回默认衣柜基础档。
- 前后双排柜体必须分别确认前排、后排进深；第一阶段只支持 `单排进深 ≤ 450mm` 的组合报价。
- 如果用户描述里出现 `双排 / 前后 / 互通 / 有门无背板 / 无门有背板` 这类床下柜关键词，直接按 `模块化儿童床 + 床下柜组合路径` 处理。
- 对 `半高床 / 高架床 + 床下柜体`：
  - 缺任意一排进深时，不正式报价。
  - 不能用前排进深默认替代后排进深。
  - 不能因为用户说了 `互通` 就默认前后同深。
  - 单排进深 `>450mm` 时，只能返回限制说明，不能拆成“床体模块价 + 常规定制衣柜”继续正式报价。

### 桌类

- 先确认长度、进深，再确认是否有柜体结构，最后补材质。

## 4. 柜体定制规则

### 4.1 投影面积

- 柜体类定制通常按投影面积计价。
- 投影面积 = 长 × 高。
- 柜体投影面积小于 `1.6㎡` 时，按 `1.6㎡` 计算。

### 4.2 进深规则

- 对定制柜体，`进深` 不是补充信息，而是价格修正条件。
- 查到基础单价后，必须继续判断进深。
- 进深尺寸按产品最大外径尺寸理解。

衣柜类常规路径：

- `450mm＜进深≤600mm`：按基础单价计算。
- `600mm＜进深≤700mm`：在基础单价上 `加价15%`。
- `进深＞700mm`：按 `前后两组柜体之和` 计算，不能继续按单组柜体直接正式报价。
- 如果正式报价命中 `600mm＜进深≤700mm`：
  - 不能只写“加价15%”
  - 必须把公式展开成：
    - `基础价格 = 投影面积 × 基础单价`
    - `超深加价 = 基础价格 × 15%`
    - `合计 = 基础价格 + 超深加价`

浅柜类常规路径：

- `进深≤450mm`：按浅柜基准路径。
- 如果明显超过浅柜基准深度，需改按对应深度路径复核，不能直接沿用原基础单价。

### 4.3 其他柜体修正项

- 异形避让部分上浮 `30%`。
- 异形避让公式：
  - `避让部分价格 = 避让部分正面投影面积 × 单价 × 1.3`
- 平板门纹理连续超过 `0.9m` 时，按对应平板门规则处理。
- `流云 / 飞瀑` 平板门纹理连续规则：
  - `≤0.9m`：仍按拼框门计价
  - `>0.9m`：按 `同材质平板门单价 - 同材质拼框门单价` 的门板差价补差
  - 表11 单独门板单价要成对理解，不能跨材质套用：
    - `单独拼框门/玻璃门/拱形门`
    - `北美樱桃木 1780 / 北美白蜡木 1780 / 北美白橡木 2080 / 乌拉圭玫瑰木 2280 / 北美黑胡桃木 2980`
    - `单独流云/飞瀑/简美/拉线/藤编/外悬条条推拉门`
    - `北美樱桃木 2380 / 北美白蜡木 2380 / 北美白橡木 2980 / 乌拉圭玫瑰木 2280 / 北美黑胡桃木 3880`
  - 同材质纹理连续示例：
    - `北美黑胡桃木流云衣柜` 做 `1.2m` 纹理连续
    - 基础柜体单价 `8680`
    - 门板差价 `3880 - 2980 = 900`
    - 调整后柜体单价 `8680 + 900 = 9580`
    - 若尺寸 `1.8 × 2.2 × 0.6m`
    - 投影面积 `1.8 × 2.2 = 3.96㎡`
    - 正式报价 `3.96 × 9580 = 37936.8`
  - 这里不能误用不同材质门板示例里的 `1780`
- 衣柜仅门板换材质时，按门板差价补差：
  - `柜体单价 = 同材质拼框门柜体单价 + (目标门板单价 - 柜体材质拼框门板单价)`
  - 这里只补 `门板差价`，不能直接拿两种材质的整柜基础单价相减
  - 例如 `北美白橡木流云衣柜 + 北美黑胡桃木流云平板门`
  - 应算 `3880 - 2980 = 900`
  - 不能算 `8680 - 6880 = 1800`
  - 原始规则示例：
    - `樱桃木柜体 + 黑胡桃平板柜门`
    - `柜体单价 = 5480 + (3880 - 1780) = 7580`
- 非见光面为乌拉圭玫瑰木时，按外露材质整柜折减：
  - `外露北美黑胡桃木：整柜价 × 0.85`
  - `外露北美白橡木：整柜价 × 0.90`
  - `外露北美樱桃木 / 北美白蜡木：整柜价 × 0.95`
  - 这是 `整柜按外露材质基础单价整体折减`
  - 不能改写成“乌拉圭玫瑰木柜体 + 外露材质门板补差”
  - 原始规则示例：
    - `流云衣柜 1.8 × 2.2 × 8680 × 0.85 = 29216.88`

### 4.4 专项结构规则

- 双面门柜体：
  - 按投影面积计价
  - 使用原始规则 `表5` 的双面门专项单价
  - 先确认两边门型组合，再取对应专项单价
  - 不要说“目录里没有双面门专项单价表”
  - 不要默认退回普通单面门柜体路径
- 操作空区 / 电视柜空区等带背板区域：
  - 带背板空区部分按原始规则 `表6` 单价单独计价
  - 优先先确认空区带背板区域的宽和高
  - 不先退回普通柜体门型追问
- 钻石柜：
  - 先确认 `钻石柜部分与旁边柜体是否同类型`
  - 若同类型（都开放或都带门），按对应长度 × 高度 × 对应柜体单价
  - 若不同类型，按图示标注长度 × 高度 × 对应柜体单价，计入组合柜体
  - 所以对外只追问这一个关键结构条件，不要泛化回普通柜体门型选择
- 背板外露时，按对应背板或加价规则处理。
- 窄边风格拆装柜体里，如果 `门盖牙称` 且要 `盖顶挡条`，最少需要预留 `15mm`。
- `直角圆边-窄边矮柜 / 高柜` 这一组节点，当前已在人工复盘层确认存在 `凹槽内退尺寸` 约束；可稳定直读到的节点尺寸包括 `p48` 的 `12 / 12 / 8`、`p49` 上节点的 `20 / 8 / 12`、`p49` 下节点的 `12 / 6 / 8`。
- 上面这组 `12 / 12 / 8`、`20 / 8 / 12`、`12 / 6 / 8` 目前只作为 `结构复盘提示` 使用，不直接当成新的硬性报价公式；其中 `20mm` 更倾向于顶挡条可见高度/宽度相关尺寸，但术语口径还没完全锁定。

## 5. 儿童房规则

- 儿童床要先区分成品和定制。
- 儿童床要先区分具体床型，不要只按“儿童床”直接正式报价。
- 儿童床、上下床、伴床、高架床、榻榻米、书桌柜都走儿童房专项路径。
- 儿童床的成品价和定制价不能混算。
- 模块化儿童床的补问顺序固定为：`床形态 > 上层出入方式 > 床垫尺寸 > 下层结构 > 材质 > 围栏样式 > 围栏尺寸/梯子尺寸`。
- 如果用户说的是 `经典护栏款 / 经典护栏 / 护栏经典款` 这类非标准围栏名称，必须先追问标准围栏名称。
- 在 `半高床 / 高架床 + 床下双排柜体` 场景里，如果同时缺标准围栏和前后柜体进深，可以放在同一轮一起追问。
- 上铺或高架床床垫宽度大于 `1.2m` 时，当前不能正式报价；要明确说明限制原因是 `上铺或高架床床垫宽度需不大于 1.2m`。

## 6. 桌类和其他单品

- 桌类优先区分标准品和定制。
- 非标桌类通常按比例或专项规则处理，不直接硬套目录标准尺寸价。
- 其他单品若已命中明确产品编号或产品名，直接沿该产品索引路径继续报价。

## 6.1 成人床专项

- 成人床如果已经命中明确产品名，优先按具体产品报价，不要退回让客户二选一“成品还是定制”。
- 成人床尺寸规则：
  - `1.2 米及以下` 按 `1.2 米` 价格
  - `1.35 米以上` 按 `1.5 米` 价格
  - `1.9 米长` 按 `2 米` 价格
- 如果目录里没有 `1.2 米` 标准价：
  - `1.2 米价格 = 1.5 米价格 - (1.8 米价格 - 1.5 米价格)`
- 超大床价格：
  - 以该床 `1.5 米` 价格为基础
  - 按修改后的长边按比例计算
  - 示例：
    - `经典箱体床 北美黑胡桃木 2 × 2 米`
    - `12800 ÷ 1.5 × 2 = 17066.67`
- 箱体床 / 架式床加高：
  - 不区分具体加高多少
  - 统一按整床价格 `+15%`
- 如果用户只说泛称 `架式床 / 箱体床`，还没有明确具体款式时：
  - 应先追问具体款式
  - 不先问“成品还是定制”

## 6.2 冰箱柜 / 无底板柜专项

- 冰箱柜、无底板柜不能直接套普通柜体投影面积总价。
- 正确结构是：
  - 上部柜体按投影面积计价
  - 下部侧板按侧板面积计价
- 如果用户只给了总长、总高、总进深，但没给 `冰箱净高` 或 `上柜高度`：
  - 先追问这个关键尺寸
  - 不先正式报价

## 7. 正式报价输出要求

正式报价必须包含：

- 产品名称
- 已确认条件
- 计价方式
- 计算过程
- 分项小计
- 最终正式报价

如果含有加价项，计算过程里还必须包含：

- 基础价格
- 加价金额
- 合计公式

不要：

- 只给总价
- 只给一句括号公式
- 只写“加价15%”但不解释最后为什么是这个金额
- 暴露 sheet、单元格、表号、源码路径

## 8. 维护规则

- 运行时只读取 `data/current/`。
- 历史版本保存在 `data/versions/`。
- 新版价格和规则优先从新的 `xlsx/docx` 导入。
- 日常维护优先只运行：

```bash
python3 ~/.openclaw/skills/liangqin-pricing/scripts/update_release.py
```

- 只有特殊排查时，才单独运行内部脚本。
