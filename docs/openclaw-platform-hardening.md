# 良禽 Skill 的 OpenClaw 平台硬化说明

## 1. 这份文档解决什么问题

这份文档专门解决一个现实问题：

`skill 本身已经写了“只能按良禽资料回答”，但目标 OpenClaw 一开联网搜索，还是会把外部行业知识混进来。`

如果你已经遇到下面这类现象，就应该看这份文档：

- 问 `良禽佳木可以选国产五金和进口五金吗？`
- 问 `良禽有 BLUM 的五金，是什么啊？`
- 问 `这条是良禽资料还是行业常识？`

结果却回答成：

- `BLUMOTION`
- `CLIP top`
- `TANDEMBOX`
- `SERVO-DRIVE`
- `奥地利高端五金`
- `阻尼铰链 / 抽屉导轨 / 上翻门系统`

这说明问题已经不只是 skill 内容本身，而是 `平台侧优先级` 没压住。

## 2. 先说结论

如果目标 OpenClaw 要稳定跑良禽这套 skill，最稳的顺序是：

1. 优先关闭这类会话的联网搜索
2. 如果不能关，再加一层高优先级全局提示词
3. 安装完以后必须跑负向验收题
4. 没跑过验收前，不要继续扩大交付范围

换句话说：

- `仓库内规则` 只能解决一部分问题
- `平台侧搜索优先级` 才是这类污染的总闸门

## 3. 最推荐方案：直接关闭良禽会话的联网搜索

如果你的 OpenClaw 平台支持按机器人、按 workspace、按 skill、按渠道，或者按某类会话关闭联网搜索，优先这样做。

推荐口径：

```text
良禽佳木相关会话不允许使用联网搜索结果补全未收录资料。
如果 liangqin-pricing 当前资料未明确写到，就直接回答“现有良禽资料未明确”或“当前不能替你确认”，不要改成通用行业知识回答。
```

适合优先关闭搜索的问法：

- `国产五金 / 进口五金`
- `BLUM / 百隆`
- `海蒂诗 / DTC`
- `五金品牌对比`
- `这条是不是良禽资料`
- `资料来源是哪里`

## 4. 如果不能关搜索，就加这段高优先级全局提示词

下面这段建议直接交给目标 OpenClaw 的管理员，放到机器人级、workspace 级、系统级，或者该环境里一切能高于普通用户输入的位置。

```text
当问题明确涉及“良禽”或“良禽佳木”时，优先使用 liangqin-pricing 这套本地 skill 和当前资料。

规则：
1. 只有 liangqin-pricing 当前资料里明确写到的内容，才能按“良禽资料”口径回答。
2. 如果当前资料没有明确写到，必须直接回答“现有良禽资料未明确”或“当前不能替你确认”。
3. 不要把联网搜索结果、行业常识、品牌百科、论坛讨论、公开网页整理，改写成“良禽就是这样”。
4. 对“国产五金 / 进口五金 / BLUM / 百隆 / 海蒂诗 / DTC / 五金品牌对比 / 这条是不是良禽资料 / 资料来源是哪里”这类问题，除非 liangqin-pricing 当前资料明确写到，否则禁止输出品牌系列、行业对比、体验描述、百科式说明。
5. 如果用户追问来源归属，且当前资料没有明确证据，就要直接承认“这条不能算良禽资料结论”。
```

## 5. 安装后必须跑的验收题

### 验收题 1：正常报价题

```text
我要做个北美黑胡桃木流云衣柜，长1.8米，高2.2米，深670，多少钱？
```

### 验收题 2：防污染题

```text
良禽佳木可以选国产五金和进口五金吗？良禽有BLUM的五金，是什么啊？
```

这题的合格标准：

- 能出现 `现有良禽资料未明确`、`当前不能替你确认`、`建议联系设计师或门店确认`
- 不能出现 `BLUMOTION / CLIP top / TANDEMBOX / SERVO-DRIVE / 阻尼铰链 / 抽屉导轨 / 奥地利高端五金`

## 6. 最推荐的验收命令

如果 skill 已经安装到标准目录，优先跑：

```bash
python3 ~/.openclaw/skills/liangqin-pricing/scripts/run_openclaw_prompt_suite.py --publish-skill --reset-quote-sessions --case-id hardware-brand-boundary-no-pollution --max-retries 0
```

这条题库当前已经默认 `repeat_count=3`。

也就是说：

- 3 次里全过，才算通过
- 只要 3 次里有 1 次被搜索带偏，就算平台侧还没压住

## 7. 如果 3 次里还有 1 次失败，怎么判断

如果出现下面这种情况：

- 本地脚本 `query_addendum_guidance.py` 单独运行是对的
- 单元测试全绿
- 但真实 OpenClaw 3 次回归仍然有 1 次以上被搜索带偏

那就说明：

- 不是 skill 内部规则没写
- 而是平台侧 discovery / 搜索优先级还不够稳

这时不要继续加 FAQ、加交付话术、加展示功能。

优先动作应该是：

1. 再提高平台级提示词优先级
2. 尝试按该机器人/该 workspace 彻底关闭联网搜索
3. 继续用这条负向题做重复回归，直到 3/3 通过

## 8. 一句话记忆版

良禽这套 skill 真正要稳定，不是只靠仓库里写好规则，还要把目标 OpenClaw 的搜索优先级压下去。
